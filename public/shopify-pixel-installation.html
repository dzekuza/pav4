üöÄ Installation Methods

Method 1: Shopify Theme Customization (Recommended)

  1. Go to your Shopify Admin ‚Üí Online Store ‚Üí Themes
  2. Click "Actions" ‚Üí "Edit code" on your active theme
  3. Navigate to "Layout" ‚Üí "theme.liquid"
  4. Add this code just before the closing </head> tag:
<script> // ipick.io Pixel Tracking - No Shopify Apps Required window.ipickConfig = { businessId: '2', // Replace with your business ID debug: false // Set to true for debugging }; </script> <script src="https://pavlo4.netlify.app/shopify-pixel-only-tracker.js"></script>

Method 2: Google Tag Manager

  1. Install Google Tag Manager on your Shopify store
  2. Create a new Custom HTML tag
  3. Add the script code from Method 1
  4. Set trigger to fire on all pages

Method 3: URL Parameter Tracking

If you can't modify your theme, you can track via URL parameters:

// Add this to your product links: https://yourstore.com/products/product-name?business_id=2&utm_source=ipick // Or add to your checkout links: https://yourstore.com/checkout?business_id=2&utm_source=ipick

üîß Configuration

Business ID Setup

The pixel automatically detects your business ID from:

  1. Manual Config: Set in window.ipickConfig.businessId
  2. URL Parameters: ?business_id=2 or ?utm_source=2
  3. Referrer: From the referring page URL
  4. Default: Falls back to business ID '1'

Debug Mode

<script> window.ipickConfig = { businessId: '2', debug: true // Enable console logging }; </script>

üîç Troubleshooting

Common Issues:

  • Script not loading: Check if the URL is accessible
  • CORS errors: Normal, script uses fallback methods
  • No events tracked: Enable debug mode and check console
  • Wrong business ID: Verify URL parameters or manual config
  • Checkout not tracking: Check CSP restrictions, script uses fallbacks

Debug Mode:

<script> window.ipickConfig = { businessId: '2', debug: true // Enable detailed logging }; </script>

üîÑ Fallback Methods

The pixel uses multiple methods to ensure tracking works:

  1. Direct Fetch: Primary method (may fail due to CORS)
  2. Image Beacon: CORS-safe method using image requests
  3. localStorage: Stores events for later processing
  4. Retry Logic: Automatically retries failed events
